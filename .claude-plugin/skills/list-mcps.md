# List MCPs Skill

View all MCP servers registered in your Claude configuration, their status, and available tools.

## When to Use

- "What MCPs do I have installed?"
- "List all my MCP servers"
- "What tools are available?"
- "Is {tool} MCP installed?"
- "Show me the MCPs thesun created"

## How to Check

### Quick List (via Claude)

Use the `/mcp` command in Claude to see registered MCP servers.

### Detailed List (via config)

Check the Claude configuration file:

```bash
# View registered MCPs
cat ~/.claude/user-mcps.json | jq '.mcpServers | keys'

# Get full MCP configuration
cat ~/.claude/user-mcps.json | jq '.mcpServers'
```

### By Location

MCPs generated by thesun are stored in:
```
~/Scripts/mcp-servers/
├── {tool}-mcp/
│   ├── src/
│   ├── dist/
│   ├── package.json
│   └── ...
```

## Output

The config shows for each MCP:
- **command**: How Claude runs the MCP
- **args**: Command arguments (usually path to index.js)
- **env**: Environment variables passed to the MCP

Example:
```json
{
  "crowdstrike-mcp": {
    "command": "node",
    "args": ["/Users/user/Scripts/mcp-servers/crowdstrike-mcp/dist/index.js"],
    "env": {
      "CROWDSTRIKE_CLIENT_ID": "",
      "CROWDSTRIKE_CLIENT_SECRET": ""
    }
  }
}
```

## Checking MCP Health

For each registered MCP, you can verify:

```bash
# Test if MCP starts gracefully (should not crash without credentials)
node /path/to/mcp/dist/index.js &
PID=$!
sleep 2
kill $PID 2>/dev/null && echo "MCP starts correctly"
```

## MCPs Created by thesun

After running thesun, MCPs are:
1. **Generated** in `~/Scripts/mcp-servers/{tool}-mcp/`
2. **Built** with `npm run build`
3. **Tested** with `npm test`
4. **Registered** in `~/.claude/user-mcps.json`

They should appear in `/mcp` after restarting Claude.

## Troubleshooting

### MCP Not Showing in /mcp

1. Check if registered: `cat ~/.claude/user-mcps.json | jq '.mcpServers'`
2. Check if dist exists: `ls ~/Scripts/mcp-servers/{tool}-mcp/dist/index.js`
3. Check if starts gracefully: `node dist/index.js` (should NOT crash)
4. Restart Claude

### MCP Shows But No Tools

1. Check credentials are set in env
2. Try calling a tool - error message will explain what's needed
