# Optimization Principles - MANDATORY

**These principles are NON-NEGOTIABLE. Every operation in thesun and every MCP it generates MUST follow them.**

## Core Rule: NEVER Sequential When Parallel Is Possible

```
❌ FORBIDDEN                          ✅ REQUIRED
─────────────────────────────────     ─────────────────────────────────
for (const mcp of mcps) {             await Promise.all(mcps.map(mcp =>
  await analyze(mcp);                   analyze(mcp)
}                                     ));

for (const check of checks) {         await Promise.all([
  await runCheck(check);                runPerfCheck(),
}                                       runSecCheck(),
                                        runFeatureCheck(),
                                      ]);

await step1();                        // Only sequential if step2 DEPENDS on step1
await step2();                        // Otherwise, run in parallel
await step3();
```

## Generated MCP Requirements

Every MCP generated by thesun MUST include:

### 1. Connection Pooling (MANDATORY)
```typescript
import http from 'http';
import https from 'https';

const httpAgent = new http.Agent({
  keepAlive: true,
  maxSockets: 50,
  maxFreeSockets: 10,
  timeout: 60000,
});

const httpsAgent = new https.Agent({
  keepAlive: true,
  maxSockets: 50,
  maxFreeSockets: 10,
  timeout: 60000,
});

// Use in axios/fetch
const client = axios.create({
  httpAgent,
  httpsAgent,
});
```

### 2. Singleton Client Pattern (MANDATORY)
```typescript
let client: ApiClient | null = null;

function getClient(): ApiClient {
  if (!client) {
    client = new ApiClient({ /* with connection pooling */ });
  }
  return client;
}
```

### 3. Token Caching (MANDATORY)
```typescript
interface TokenCache {
  token: string;
  expiresAt: number;
}

let tokenCache: TokenCache | null = null;

async function getToken(): Promise<string> {
  if (tokenCache && Date.now() < tokenCache.expiresAt - 60000) {
    return tokenCache.token;
  }
  const { token, expires_in } = await authenticate();
  tokenCache = {
    token,
    expiresAt: Date.now() + (expires_in * 1000),
  };
  return token;
}
```

### 4. Response Caching for Static Data (MANDATORY where applicable)
```typescript
const cache = new Map<string, { data: unknown; expiresAt: number }>();

async function getCached<T>(
  key: string,
  fetcher: () => Promise<T>,
  ttlMs = 300000
): Promise<T> {
  const cached = cache.get(key);
  if (cached && Date.now() < cached.expiresAt) {
    return cached.data as T;
  }
  const data = await fetcher();
  cache.set(key, { data, expiresAt: Date.now() + ttlMs });
  return data;
}
```

### 5. Parallel Batch Operations (MANDATORY for bulk tools)
```typescript
async function batchOperation<T, R>(
  items: T[],
  operation: (item: T) => Promise<R>,
  concurrency = 10
): Promise<R[]> {
  const results: R[] = [];
  for (let i = 0; i < items.length; i += concurrency) {
    const batch = items.slice(i, i + concurrency);
    const batchResults = await Promise.all(batch.map(operation));
    results.push(...batchResults);
  }
  return results;
}

// Usage in tools
case "tool_bulk_operation":
  return batchOperation(args.ids, id => fetchItem(id));
```

### 6. NO Shell Spawning (FORBIDDEN)
```typescript
// ❌ ABSOLUTELY FORBIDDEN - spawning shell processes (child_process, curl, wget, etc.)
// This creates 100x+ overhead compared to native HTTP

// ✅ REQUIRED: Native HTTP client
const response = await client.get('/data');
```

## Validation Checklist

Before any MCP is considered complete, verify:

| Check | Required | How to Verify |
|-------|----------|---------------|
| Connection pooling | ✅ | `grep -r "keepAlive" src/` |
| Singleton client | ✅ | `grep -r "let client.*null" src/` |
| Token caching | ✅ | `grep -r "tokenCache\|expiresAt" src/` |
| No shell spawning | ✅ | `grep -r "child_process\|spawn(" src/` returns nothing |
| Batch operations | ✅ (if bulk tools) | `grep -r "Promise.all" src/` |
| Response caching | ✅ (if static data) | `grep -r "cache\|Cache" src/` |

## Performance Targets

| Metric | Target | Unacceptable |
|--------|--------|--------------|
| Tool call latency | < 500ms | > 2s |
| Bulk operation (100 items) | < 5s | > 30s |
| Startup time | < 1s | > 3s |
| Memory usage | < 100MB | > 500MB |

## When Updating MCPs

The `update-mcps` skill MUST:

1. **First**: Check for optimization anti-patterns
2. **Fix**: Any violations of these principles
3. **Measure**: Before/after performance
4. **Report**: Performance improvements

Performance optimization is **CRITICAL priority** - same level as security vulnerabilities.

## Enforcement

These principles are enforced by:
1. `mcp-builder` agent - Generates optimized code from start
2. `validator` - Checks for optimization patterns
3. `mcp-updater` - Fixes violations in existing MCPs
4. Code review - Rejects MCPs that violate principles
